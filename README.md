# 入庫一括（nyuko-ikkatsu）

ラクマートで発注した商品が自社に到着したときに使う、入庫処理一括アプリです。

ブラウザ内でファイルを読み取り、以下の3ファイルを生成します。

- `NE更新.csv`
- `kintone更新.csv`
- `入庫リスト.xlsx`

ファイルはサーバーへ送信しません。GitHub Pagesで静的公開できます。

## 入力ファイル

### 1. ラクマート配送依頼書 `P~.xlsx`

- 複数ファイル対応
- `梱包リスト` シートを使用
- `箱詰め備考` 列から `●商品コード▲MMDD-数量` を抽出

### 2. オーダー状況CSV

kintoneビューからダウンロードしたCSVです。

必要列:

- `商品番号`
- `商品名`
- `オーダー1`〜`オーダー5`
- `RM1`〜`RM5`

### 3. 商品情報.csv

必要列:

- `商品番号`
- `出荷時商品名`
- `階数`

## 消し込みルール

ラクマート配送依頼書から抽出した `MMDD-数量` を、kintoneの `オーダー1`〜`オーダー5` と照合します。

一致条件は以下です。

- 完全一致: `0416-500`
- 注記付きも一致: `0416-500setRM`, `0416-500空`, `0416-500保管`
- 数字が続くものは不一致: `0416-5000`

消し込み後、残ったオーダーとRMは左詰めします。

## ローカル起動

```bash
npm install
npm run dev
```

ブラウザで `http://localhost:3000` を開きます。

## 静的ビルド

```bash
npm run build
```

`out/` に静的ファイルが出力されます。

## GitHub Pages公開

このリポジトリには GitHub Actions の Pages デプロイ設定を同梱しています。

1. GitHubに `nyuko-ikkatsu` リポジトリを作成
2. このプロジェクトをpush
3. GitHubの `Settings > Pages` で Source を `GitHub Actions` にする
4. `main` ブランチにpushすると自動公開

公開URL例:

```text
https://ユーザー名.github.io/nyuko-ikkatsu/
```
